# AI Agent Guide - adk-utils-example

## Project Description
This is a Next.js 16 (App Router) project with TypeScript that demonstrates the use of the `@yagolopez/adk-utils` library. It's a starter/example application created with `create-next-app`.

## Tech Stack
- **Framework**: Next.js 16.1.6 (App Router)
- **Language**: TypeScript 5
- **UI**: React 19.2.3
- **Styles**: Tailwind CSS 4
- **Main Library**: @yagolopez/adk-utils
- **Linting**: ESLint 9 with eslint-config-next

## Project Structure
```
adk-utils-example/
├── app/                    # Next.js App Router
│   ├── layout.tsx         # Root layout with Geist fonts
│   ├── page.tsx           # Main page
│   ├── globals.css        # Global styles
│   └── favicon.ico        # Icon
├── public/                # Static files
├── .next/                 # Build output (ignore)
└── node_modules/          # Dependencies (ignore)
```

## Code Conventions

### TypeScript
- **Strict mode**: `strict: true` enabled
- **Target**: ES2017
- **JSX**: react-jsx (no need to import React in every file)
- **Path aliases**: Use `@/*` for imports from root

### React/Next.js
- **Components**: Use React Server Components by default
- **Client Components**: Explicitly mark with `'use client'` when needed
- **Exports**: Use `export default` for pages and layouts
- **Metadata**: Define metadata in layouts using Next.js `Metadata` object

### Styles
- **Framework**: Tailwind CSS 4
- **Classes**: Use Tailwind utility classes
- **Dark mode**: Supported with `dark:` class
- **Fonts**: Geist Sans and Geist Mono configured as CSS variables

### Naming Conventions
- **Components**: PascalCase (e.g., `Home`, `RootLayout`)
- **Files**: kebab-case or camelCase following Next.js conventions
- **CSS Variables**: kebab-case with `--` prefix (e.g., `--font-geist-sans`)

## AI Rules

### When Creating Code
1. **Always use TypeScript** - Don't use plain JavaScript
2. **Type properly** - Define explicit types, use `Metadata`, `NextConfig`, etc.
3. **Server Components by default** - Only use `'use client'` when you need:
   - React Hooks (useState, useEffect, etc.)
   - Event handlers
   - Browser APIs
   - Context providers
4. **Next.js Imports** - Use optimized components:
   - `next/image` for images
   - `next/link` for navigation
   - `next/font` for fonts
5. **Accessibility** - Include `alt`, `aria-*` attributes when relevant

### When Modifying Code
1. **Maintain existing style** - Follow project conventions
2. **Preserve configurations** - Don't change `tsconfig.json` or `next.config.ts` without reason
3. **Update imports** - Keep consistency with `@/*` path aliases

### When Answering Questions
1. **Next.js 16 App Router context** - Use the latest documentation
2. **React 19** - Take into account new features like React Server Components
3. **@yagolopez/adk-utils** - Reference this library when relevant

### When Debugging
1. Check TypeScript errors first
2. Review if the component needs `'use client'`
3. Confirm Next.js imports are correct
4. Verify App Router structure (not Pages Router)
